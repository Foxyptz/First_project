ОПИСАНИЕ ПРОЕКТА

Из «Бета-Банка» стали уходить клиенты. Банковские маркетологи посчитали: сохранять текущих клиентов дешевле, чем привлекать новых.
Нужно спрогнозировать, уйдёт клиент из банка в ближайшее время или нет на основе исторических данных о поведении клиентов и расторжении договоров с банком.
Требуется построить модель с предельно большим значением F1-меры (нужно довести метрику до 0.59). Дополнительно требуется измерять AUC-ROC и сравнивать её значение с F1-мерой.

Данные находятся в файле /datasets/Churn.csv .

ПРИЗНАКИ

RowNumber — индекс строки в данных
CustomerId — уникальный идентификатор клиента
Surname — фамилия
CreditScore — кредитный рейтинг
Geography — страна проживания
Gender — пол
Age — возраст
Tenure — сколько лет человек является клиентом банка
Balance — баланс на счёте
NumOfProducts — количество продуктов банка, используемых клиентом
HasCrCard — наличие кредитной карты
IsActiveMember — активность клиента
EstimatedSalary — предполагаемая зарплата

ЦЕЛЕВОЙ ПРИЗНАК

Exited — факт ухода клиента

ЭТАПЫ

1. Подготовка данных, замена категориальных данных числовыми техникой прямого кодирования, а также масштабирование с использованием StandartScaler.
2. Обучение моделей без учета дисбаланса данных: логистическая регрессия, решающее дерево, случайный лес.
3. Борьба с дисбалансом апсэмплингом, повторное обучение моделей, затем то же самое, но с использованием даунсэмплинга. Наилучший результат показал upsampling и модель случайного леса.
4. Тестирование лучшей модели, необходимые значения метрик достигнуты.
